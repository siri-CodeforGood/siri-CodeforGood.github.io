Why This Work Matters

Energy demand spikes put stress on the grid. Utilities need customers to reduce usage at peak times — but the systems connecting utilities, devices, and customers are slow, manual, and fragmented. My work at Meltek is to fix that.

Problem: The entire production database ran on MySQL. Scaling, cost, and feature requirements demanded PostgreSQL. A failed migration would mean data loss, downtime, or corrupted production state.
Solution: Designed and executed a comprehensive migration strategy including new schema design, data validation scripts, interval data migration with deduplication logic, and Alembic CI/CD pipeline for ongoing schema management. Environment separation (dev/prod) established during the process.

Problem: Legacy Consumption Logic Apps had reliability, cost, and scalability issues. The Meltek-GBC Logic App and several others were creating maintenance burden and unnecessary Azure spend.

Production Reliability Engineering
Diagnosing and fixing critical production incidents across DB, email, SMS, and cloud routing
Production Ops
Problem: Multiple critical production incidents: FunctionApp timeouts from DB connection exhaustion (PROD-4337, PROD-4352), Microsoft domain email delivery failure on July 17th, Twilio events routed to dev instead of prod, phone-number lookup hitting dev backend, and unnecessary backend autoscaling from missing DB connection recycling in enrollment flows.
Solution — layered fixes:
  • Implemented PostgreSQL async query helpers with connection pooling — eliminated FunctionApp timeouts
  • Separate pooling in postgres.py for generic vs webhook use cases
  • get_postgres_connection() returns pool with connection — updated all interval recurring flows
  • Debugged SendGrid config for Yahoo/AOL & Microsoft domain delivery failures
  • Fixed Twilio event routing — dev traffic now properly isolated from prod
  • Fixed phone-number lookup events sent to dev instead of prod
  • Added DB connection recycling to bulk & enrollment flows — stopped unnecessary container scaling
Outcome: All production incidents resolved. DB connection exhaustion eliminated. Email delivery restored for all domains. Autoscaling normalized. Monitoring in place to prevent recurrence.
